# Prompt: สร้างปลั๊กอิน EssentialsBE สำหรับ Endstone (Bedrock)

> คัดลอกทั้งหมดด้านล่างนี้ไปวางให้ AI ได้เลย

---

สร้างปลั๊กอิน **Endstone** (Python) ชื่อ `EssentialsBE` เลียนแบบฟีเจอร์ของ EssentialsX ฝั่ง Java Edition สำหรับใช้บน Minecraft Bedrock Dedicated Server ตามข้อกำหนดต่อไปนี้ทั้งหมด

## 0. ข้อกำหนดพื้นฐาน

- เขียนด้วย Python ตามโครงสร้างปลั๊กอิน Endstone เวอร์ชันล่าสุด (อ้างอิง API จากเอกสารทางการ https://endstone.dev ถ้าไม่แน่ใจ API ตัวไหนให้ตรวจสอบจากเอกสารก่อน ห้ามเดา)
- โครงสร้างโปรเจกต์ครบ: `pyproject.toml` พร้อม entry point ของ endstone, class หลักสืบทอดจาก `endstone.plugin.Plugin` และประกาศ `commands` / `permissions` เป็น class attributes
- แยกโค้ดเป็นโมดูล: `commands/` (แยกไฟล์ตามหมวด), `managers/` (home, warp, tpa, kit, playerdata), `utils/`
- เก็บข้อมูลผู้เล่นเป็น **JSON ต่อคน**: `plugins/essentials_be/userdata/<xuid>.json` (เก็บ homes, จุด /back, cooldown ของ kit, สถานะ tptoggle)
- ไฟล์ตั้งค่า: `config.yml`, `kits.yml`, `warps.json`, `spawn.json`
- ข้อความทั้งหมดอยู่ใน `messages/th.yml` และ `messages/en.yml` เลือกภาษาได้ใน config (ค่าเริ่มต้น: th) รองรับ placeholder เช่น `{player}`, `{home}`, `{seconds}` และโค้ดสี `§`
- ทุกคำสั่งตรวจ permission รูปแบบ `essentials.<command>` โดย OP ได้ทุกสิทธิ์เป็นค่าเริ่มต้น
- **Form UI**: ใช้ `ActionForm` / `MessageForm` / `ModalForm` ของ Endstone สำหรับเมนูเลือก (home, warp, kit, ลบ home) โดยมีกติกากลาง: (1) เปิด/ปิดทั้งระบบได้ใน config ด้วย `use-forms: true` ถ้าปิดให้ fallback เป็นข้อความ (2) การวาร์ปที่เกิดจากการกดปุ่มในฟอร์มต้องผ่าน warmup/cooldown เหมือนพิมพ์คำสั่งปกติ (3) ถ้าผู้เล่นปิดฟอร์มโดยไม่เลือก ต้องไม่เกิดอะไรและไม่มี error (4) เขียนตัวสร้างฟอร์มเป็น helper กลางใน `utils/forms.py` ไม่เขียนซ้ำทุกคำสั่ง

## 1. หมวด Teleport / Home / Warp / Spawn

### Home
- `/home` (ไม่ใส่ชื่อ) — เปิด **Form UI (ActionForm)** แสดง home ทั้งหมดเป็นปุ่ม กดแล้ววาร์ปทันที (ถ้ามี home เดียวให้วาร์ปเลยไม่ต้องเปิดฟอร์ม) / `/home <ชื่อ>` — วาร์ปตรงโดยไม่เปิดฟอร์ม
- `/sethome [ชื่อ]` — ตั้ง home (ค่าเริ่มต้นชื่อ `home`)
- `/delhome` (ไม่ใส่ชื่อ) — เปิด Form ให้เลือก home ที่จะลบ + MessageForm ยืนยันก่อนลบ / `/delhome <ชื่อ>` — ลบตรง ๆ
- `/homes` — แสดงรายชื่อ home ทั้งหมดเป็นข้อความ (สำหรับดูเฉย ๆ)
- จำกัดจำนวน home ต่อคนผ่าน permission `essentials.sethome.multiple.<n>` (เช่น `.multiple.3` = 3 อัน) ถ้าไม่มี permission พิเศษให้ใช้ค่า `default-homes` ใน config
- เก็บพิกัด + dimension (overworld/nether/end) และวาร์ปข้ามมิติได้ถูกต้อง

### Warp / Spawn
- `/warp` (ไม่ใส่ชื่อ) — เปิด Form UI แสดงเฉพาะ warp ที่ผู้เล่นมีสิทธิ์ใช้เป็นปุ่ม กดแล้ววาร์ป / `/warp <ชื่อ>` — วาร์ปตรง
- `/setwarp <ชื่อ>`, `/delwarp <ชื่อ>` — สำหรับแอดมิน (`essentials.setwarp`, `essentials.delwarp`)
- รองรับ permission ต่อ warp: `essentials.warps.<ชื่อ>` (เปิด/ปิดโหมดนี้ใน config ด้วย `per-warp-permission`)
- `/spawn` — วาร์ปไปจุด spawn ของเซิร์ฟเวอร์, `/setspawn` — ตั้งจุด spawn (แอดมิน)

### TPA
- `/tpa` (ไม่ใส่ชื่อ) — เปิด Form UI แสดงรายชื่อผู้เล่นออนไลน์เป็นปุ่ม (ไม่แสดงตัวเองและคนที่ปิด tptoggle) กดเพื่อส่งคำขอ / `/tpa <ผู้เล่น>` — ส่งคำขอตรง
- `/tpahere` — เหมือน `/tpa` ทุกอย่างแต่เป็นการขอให้อีกฝ่ายวาร์ปมาหา (มีฟอร์มเลือกผู้เล่นเช่นกัน)
- **ฝั่งผู้รับคำขอ**: เมื่อมีคำขอเข้ามา เปิด `MessageForm` ให้ทันที บอกว่าใครขอและขอแบบไหน (มาหา/ให้เราไป) พร้อมปุ่ม "ยอมรับ" / "ปฏิเสธ" — ถ้าผู้รับปิดฟอร์มโดยไม่กด คำขอยังค้างอยู่จนหมดเวลา และยังใช้ `/tpaccept` `/tpdeny` พิมพ์เองได้ตามปกติ (เปิด/ปิดฟอร์มเด้งอัตโนมัตินี้ได้ใน config ด้วย `tpa-request-form: true` เผื่อบางเซิร์ฟไม่อยากให้ฟอร์มเด้งกลางการต่อสู้)
- `/tpaccept`, `/tpdeny`, `/tpacancel` — ทำงานแบบพิมพ์คำสั่งได้เสมอ ถ้ามีหลายคำขอค้างอยู่ `/tpaccept` โดยไม่ระบุชื่อให้เปิดฟอร์มเลือกว่าจะรับของใคร
- `/tptoggle` — สลับเปิด/ปิดการรับคำขอวาร์ป พร้อมข้อความบอกสถานะปัจจุบัน (สถานะบันทึกลง userdata คงอยู่หลังรีสตาร์ต)
- คำขอหมดอายุอัตโนมัติ (ค่าเริ่มต้น 60 วินาที ตั้งได้ใน config) และมีข้อความแจ้งทั้งสองฝ่ายทุกขั้นตอน: ส่งแล้ว / ได้รับคำขอ / ยอมรับ / ปฏิเสธ / หมดเวลา / อีกฝ่ายออกจากเกม

### อื่น ๆ
- `/back` — กลับไปจุดก่อนวาร์ปครั้งล่าสุด และจุดตายล่าสุด (บันทึกเมื่อผู้เล่นตายผ่าน event และก่อนการวาร์ปทุกครั้งของปลั๊กอิน) permission แยก: `essentials.back` และ `essentials.back.ondeath`
- `/rtp` — สุ่มวาร์ปในรัศมีที่ตั้งใน config (min/max radius) พร้อมตรวจหาจุดปลอดภัย: ไม่ลงลาวา น้ำ ช่องว่าง หรือใต้ดิน มี cooldown ต่อคน
- `/tppos <x> <y> <z>` — วาร์ปไปพิกัด (แอดมิน)
- `/top` — วาร์ปขึ้นบล็อกบนสุดที่ยืนได้ ณ จุดปัจจุบัน

### Warmup / Cooldown (ใช้กับทุกการวาร์ปของปลั๊กอิน)
- `teleport-warmup` (วินาที): ต้องยืนนิ่งก่อนวาร์ป ถ้าขยับ/โดนตี ให้ยกเลิกพร้อมข้อความแจ้ง
- `teleport-cooldown` (วินาที): กันสแปมวาร์ป
- ข้าม warmup/cooldown ได้ด้วย `essentials.teleport.timer.bypass`
- ใช้ scheduler ของ Endstone เท่านั้น **ห้ามใช้ time.sleep บล็อกเธรดหลัก**

## 2. หมวด Kits

- `/kit` (ไม่ใส่ชื่อ) — เปิด Form UI แสดง kit ที่ผู้เล่นมีสิทธิ์ใช้เป็นปุ่ม พร้อมบอกสถานะ cooldown ในชื่อปุ่ม (เช่น "daily §c(เหลือ 2 ชม.)") kit ที่ติด cooldown กดแล้วให้แจ้งเวลาที่เหลือแทนการแจกของ
- `/kit <ชื่อ>` — รับ kit ตรง ๆ
- นิยาม kit ทั้งหมดใน `kits.yml` ต่อ kit กำหนดได้: รายการไอเทม (item id ของ Bedrock, จำนวน, ชื่อ custom ถ้ามี), `cooldown` (วินาที, `-1` = ใช้ครั้งเดียวตลอดกาล), `auto-equip` เกราะอัตโนมัติ (ถ้า API รองรับ)
- permission ต่อ kit: `essentials.kits.<ชื่อ>`
- ถ้าช่องเก็บของเต็ม ให้ดรอปของที่เหลือลงพื้นพร้อมแจ้งเตือน
- บันทึกเวลา cooldown ลง userdata ให้คงอยู่แม้รีสตาร์ตเซิร์ฟเวอร์
- แถมตัวอย่าง kit ใน kits.yml มา 2 ชุด (starter, daily)

## 3. การจัดการคำสั่งที่ทับกับ vanilla

- **ห้ามพยายาม override คำสั่ง vanilla โดยตรง** (ทำไม่ได้บน BDS)
- คำสั่งที่ vanilla มีอยู่แล้ว (`/tp`, `/give`, `/gamemode`, `/time`, `/weather`, `/kill`, `/spawnpoint` ฯลฯ) ให้ปล่อยใช้ของเกมตามปกติ ปลั๊กอินไม่ต้องทำซ้ำ
- คำสั่งของปลั๊กอินทุกตัวต้องตั้งชื่อไม่ชนกับ vanilla ถ้า Endstone แจ้งว่าชื่อชน ให้ลงทะเบียน alias สำรองที่ขึ้นต้นด้วย `e` เช่น `/ehome`
- ถ้าจำเป็นต้องเสริมพฤติกรรมให้คำสั่ง vanilla ให้ใช้การดักผ่าน `PlayerCommandEvent` (ตรวจข้อความคำสั่ง → cancel → ให้ปลั๊กอินจัดการแทน) โดยเขียนเป็นระบบ interceptor แยกโมดูล เปิด/ปิดได้ใน config — ในเวอร์ชันนี้ให้ใช้ดักการวาร์ปด้วย `/tp` ของ vanilla เพื่อบันทึกจุด `/back` เท่านั้น (ถ้า event อ่านได้)

## 4. คุณภาพโค้ดและ Deliverables

- บันทึกไฟล์ JSON แบบ atomic (เขียนลงไฟล์ชั่วคราวก่อนแล้ว rename) กันไฟล์พัง
- โหลด userdata ตอนผู้เล่น join, เซฟตอน quit และ autosave ทุก 5 นาที
- ใส่ try/except ครอบจุดเสี่ยงทั้งหมด และ log ผ่าน `self.logger` (ห้ามใช้ print)
- คอมเมนต์อธิบายโค้ดเป็นภาษาไทยสั้น ๆ ตามจุดสำคัญ
- สร้าง `README.md` ประกอบ: วิธีติดตั้ง, โครงสร้างไฟล์ config, **ตารางคำสั่ง + permission ทั้งหมด**
- ส่งมอบ: โค้ดครบทุกไฟล์ พร้อมโครงสร้างโฟลเดอร์ ติดตั้งได้จริงบน Endstone ทันที
